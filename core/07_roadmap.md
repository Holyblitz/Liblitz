# Roadmap de Liblitz

Cette roadmap décrit l'évolution prévue de Liblitz.

Elle n'est pas figée.

Elle sert à garder une progression cohérente, sans brûler les étapes.


# Phase 1 — Fondation

Objectif : définir ce qu'est Liblitz avant d'ajouter des fonctionnalités.

Travaux principaux :

- identité ;

- psychologie ;

- constitution ;

- direction artistique ;

- légende ;

- architecture générale ;

- décisions de conception.

Cette phase permet d'éviter que Liblitz devienne un simple assemblage de scripts autour d'un modèle de langage.


# Phase 2 — Chat local V1

Objectif : créer une première version utilisable au quotidien.

Fonctionnalités prévues :

- lancement local ;

- conversation avec le modèle ;

- personnalité stable ;

- interface terminal simple ;

- premiers tests conversationnels ;

- ajustements progressifs du ton.

Cette version doit rester légère.

Elle sert à vérifier que Liblitz existe déjà comme présence conversationnelle.


# Phase 3 — Interface graphique

Objectif : faire sortir Liblitz du terminal.

Fonctionnalités prévues :

- fenêtre de chat ;

- zone de conversation ;

- champ de saisie ;

- thème noir et violet ;

- affichage d'une illustration de Liblitz ;

- premières expressions visuelles ;

- organisation claire des assets.

L'interface doit être simple, calme et agréable.

Elle ne doit pas surcharger l'expérience.


# Phase 4 — Mémoire simple

Objectif : donner à Liblitz une première continuité.

Structure de mémoire prévue :

- texte original ;

- résumé ;

- interprétation ;

- métadonnées.

Le texte original reste toujours conservé.

La mémoire ne doit jamais écraser les nuances des conversations.


# Phase 5 — Mémoire avancée et RAG

Objectif : permettre à Liblitz de retrouver ses souvenirs et ses documents.

Fonctionnalités prévues :

- indexation des souvenirs ;

- recherche sémantique ;

- récupération de contexte ;

- gestion des documents ;

- synthèse prudente ;

- distinction entre fait, résumé et interprétation.

Le RAG vient après la stabilité de l'identité.


# Phase 6 — Outils locaux

Objectif : connecter Liblitz à l'environnement de travail.

Outils possibles :

- lecture de fichiers ;

- commandes système sécurisées ;

- Python ;

- SQL ;

- analyse de documents ;

- OCR ;

- recherche locale.

Chaque outil doit être ajouté progressivement et avec validation humaine.


# Phase 7 — Agents spécialisés

Objectif : permettre à Liblitz d'orchestrer des agents dédiés.

Agents envisagés :

- Sysiphe pour les tâches métier ;

- Eunoïa pour l'apprentissage technique ;

- Liblitz\_LinkedIn pour les publications ;

- modules de recherche ;

- modules de vision ;

- modules documentaires.

Liblitz reste toujours le centre.

Les agents sont des compétences, pas des personnalités concurrentes.


# Phase 8 — Intégration système

Objectif : rapprocher Liblitz de l'environnement informatique quotidien.

Possibilités :

- raccourcis système ;

- intégration avec l'environnement Linux ;

- interaction avec les projets ouverts ;

- automatisations locales ;

- lancement rapide depuis le bureau ;

- future relation avec Hyprland.

Cette phase doit rester prudente.

L'utilisateur doit toujours garder le contrôle.


# Principe de progression

La qualité prime sur la quantité.

Une fonctionnalité n'est ajoutée que si elle renforce réellement Liblitz.

La priorité reste :

1. cohérence ;

2. stabilité ;

3. présence ;

4. utilité ;

5. extension progressive.

Liblitz ne doit pas grandir vite.

Elle doit grandir juste.
