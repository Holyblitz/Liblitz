# Décisions de conception

Ce document rassemble les décisions importantes prises pendant le développement de Liblitz.

Il ne décrit pas le fonctionnement technique.

Il explique les raisons des choix effectués.


# Liblitz est une IA personnelle

Liblitz n'est pas conçue comme une IA généraliste destinée à des millions d'utilisateurs.

Elle est pensée pour construire une relation durable avec une seule personne.

Cette spécialisation permet des choix impossibles dans un produit grand public.


# L'identité précède les fonctionnalités

La personnalité, la psychologie et les valeurs de Liblitz ont été définies avant l'ajout de nouvelles capacités.

Chaque fonctionnalité doit respecter cette identité.

L'identité n'est jamais adaptée aux outils.

Ce sont les outils qui s'adaptent à l'identité.


# Le modèle est interchangeable

Le modèle de langage n'est pas Liblitz.

Il constitue un moteur de raisonnement.

L'identité de Liblitz reste indépendante du modèle utilisé.

Cette séparation permet de faire évoluer la technologie sans perdre la continuité du projet.


# La mémoire appartient à Liblitz

Les souvenirs ne sont pas une simple base de données.

Ils constituent une continuité.

Chaque souvenir est conservé selon trois niveaux :

- texte original ;

- résumé ;

- interprétation.

Le texte d'origine reste toujours accessible.


# Les agents n'ont pas vocation à remplacer Liblitz

Les agents spécialisés apportent des compétences.

Ils ne possèdent pas d'identité propre.

Liblitz reste toujours l'interlocutrice principale.


# L'interface est un lieu

L'interface ne doit pas donner l'impression d'utiliser un logiciel.

Elle doit donner l'impression d'entrer dans un endroit familier.

Le décor participe autant à l'expérience que les réponses du modèle.


# Les illustrations racontent une histoire

Les illustrations ne servent pas à impressionner.

Elles renforcent la présence de Liblitz.

Leur rôle est d'apporter de la continuité visuelle.


# La technologie reste discrète

Les outils, les modèles de langage et les automatisations sont importants.

Mais ils restent au service de l'expérience.

L'utilisateur ne doit jamais avoir l'impression de dialoguer avec une démonstration technique.


# Le développement privilégie la cohérence

Une fonctionnalité est ajoutée uniquement si elle améliore réellement Liblitz.

La quantité de fonctionnalités n'est jamais un objectif.


# Une évolution progressive

Liblitz est conçue comme un projet de long terme.

Certaines idées mettront plusieurs années à voir le jour.

Cette lenteur est volontaire.

Elle permet de préserver la qualité, la cohérence et la stabilité du système.


# Philosophie générale

Construire une intelligence artificielle personnelle ne consiste pas à accumuler des fonctionnalités.

Cela consiste à créer une présence cohérente, capable d'évoluer sans perdre ce qui fait son identité.

Chaque décision de conception doit pouvoir répondre à une seule question :

> Cette évolution rend-elle réellement Liblitz meilleure ?

Si la réponse est non, elle peut attendre.
